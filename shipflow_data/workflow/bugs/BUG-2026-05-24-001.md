## Summary

Same-profile Android network switching does not meet the WebView pooling acceptance criterion. Returning to a previously opened network visibly reloads and takes about 4 seconds instead of showing an already warm host.

## Reproduction

- Preconditions:
  - Install SocialGlowz Android APK from GitHub Actions / Blacksmith artifact.
  - Use the Android WebView pooling build.
- Steps:
  1. Open SocialGlowz on Android.
  2. Open a network in Profile A and wait until it loads.
  3. Switch to another network.
  4. Return to the first network.
- Expected behavior:
  - The original session host is shown from the warm pool without full `loadUrl` navigation.
  - `show_webview` should return `true` when the host exists.
- Observed behavior:
  - The first network takes about 4 seconds to reappear and visibly reloads every time.

## Impact

- The user-facing performance goal of the Android pooling chantier is not met.
- Session isolation initial manual QA passed for Profile A/Profile B flows, but the fast-switch baseline still fails.

## Initial Hypotheses

- The Vue layer may be falling back from `show_webview` to `open_webview` because Android returns `false`.
- The Kotlin host may not be retained in `sessionHosts` after hide, close, navigation, or component lifecycle changes.
- The installed APK may not include the latest pooling implementation, or the device may be in fallback single-WebView mode.
- Native bottom-bar switching and Vue-side switching may be exercising different paths.
- Current strongest hypothesis: stale APK/artifact. The copied logs do not match the current source tree because they include `reuse existing webview (switch)` and no current multi-profile mode line.

## Evidence Needed

- App debug logs copied immediately after reproducing the switch.
- If available, `adb logcat -d -s SFZ` covering app launch, first open, network switch, and return.
- Whether logs include `android-webview mode=multi-profile` or `android-webview mode=fallback-single-webview`.
- Whether return logs show a new `OPEN webview` for the previous network.
- Device model, Android version, and APK artifact/commit if available.
